<p>​​When discussing something with the client (existing PBI, new PBI, Pull request...), it is important to do it in the right place.</p><h3 class="ssw15-rteElement-H3">For a new PBI </h3><p>Create a new issue mentioning the Product Owner and&#160;the related people</p><p></p><dd class="ssw15-rteElement-FigureNormal"><img src="/SiteAssets/discuss-the-backlog/Create-Issue.png" alt="Create-Issue.png" style="margin&#58;5px;width&#58;808px;" /></dd><dd class="ssw15-rteElement-FigureNormal">Figure&#58; When adding a GitHub issue, @ mention the Product Owner and other related people so they receive a notification e.g, an email<br></dd><h3 class="ssw15-rteElement-H3">For an existing issue</h3><p>Discuss it in the existing PBI</p><div> <img src="/SiteAssets/discuss-the-backlog/existing%20issue%20discussion.png" alt="existing issue discussion.png" style="margin&#58;5px;width&#58;808px;" />&#160;</div><dd class="ssw15-rteElement-FigureNormal">Figure&#58; You can discuss an existing issue, even when it has been closed<br></dd><h3 class="ssw15-rteElement-H3">For a code issue</h3><p class="ssw15-rteElement-P">Sometimes between developers, you need some explanations about why something was done a certain way.<br></p><p class="ssw15-rteElement-P">It should be done in the&#160;Pull Request, if possible comment directly on the&#160;line of the code change </p><div><img src="/SiteAssets/discuss-the-backlog/comment%20on%20code%20in%20pull%20request.png" alt="" style="margin&#58;5px;width&#58;808px;" /><br></div><dd class="ssw15-rteElement-FigureNormal">Figure&#58; you can add a comment on a specific line of code&#160; <br></dd><h3 class="ssw15-rteElement-H3">For other&#160;topics (brainstorm ideas, general discussion...)</h3><div>You can&#58;<br> <ul><li></li><li>Create an issue and have it checked by the client before publishing it (recommended)<br></li></ul></div><ul><li>Discuss it in the discussion tab in GitHub<br></li></ul><blockquote><div><img src="/SiteAssets/discuss-the-backlog/Discussion%20Tab%20in%20GitHub.png" alt="Discussion Tab in GitHub.png" style="margin&#58;5px;width&#58;808px;" />&#160;</div></blockquote><blockquote><div><dd class="ssw15-rteElement-FigureNormal">Figure&#58; You can create discussions on your&#160;project using the Discussion tab in your GitHub repository</dd></div></blockquote><div><br></div><div><ul><li>In the team channel in Teams</li></ul><blockquote><div><img src="/SiteAssets/discuss-the-backlog/Team%20channel.png" alt="Team channel.png" style="margin&#58;5px;width&#58;808px;" />&#160;</div></blockquote><blockquote><div><dd class="ssw15-rteElement-FigureNormal">Figure&#58; You can discuss your idea in Teams on the team channel<br></dd></div></blockquote></div><ul><li>Email as a last resort</li></ul>

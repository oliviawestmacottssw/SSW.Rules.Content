​ 
<p>​When a new developer joins a project, there is often a sea of information that they need to learn right away to be productive. This includes things like: ​ 
   <br></p><ol><li>​​Who the 
         <a href="/_layouts/15/FIXUPREDIRECT.ASPX?WebId=3dfc0e07-e23a-4cbb-aac2-e778b71166a2&TermSetId=07da3ddf-0924-4cd2-a6d4-a4809ae20160&TermId=a91de77d-a8dd-40be-b61a-bd6305e7183f">Product Owner​</a> is and who the 
         <a href="/_layouts/15/FIXUPREDIRECT.ASPX?WebId=3dfc0e07-e23a-4cbb-aac2-e778b71166a2&TermSetId=07da3ddf-0924-4cd2-a6d4-a4809ae20160&TermId=c38e68bd-763b-4e19-b3ab-6083115ac1f2">Scrum Master​</a> is<br></li>
   <li>Where the backlog is<br></li><li>Where the automated builds are<br></li><li>Where the staging and production environments are</li><li>How to set up the development environment for the project</li></ol><p>Make it easy for the new developer by putting all this information in a central location like the Visual Studio dashboard.<br></p><dl class="image"><dt>

<p>The dashboard should contain:</p><ol><li><span style="line-height:1.6;">Who the <a href=/do-you-know-the-how-to-be-a-good-product-owner>Product Owner </a> is and who the <a href=/the-team-do-you-help-your-scrum-master-not-scrummaster-protect-and-serve-the-team>Scrum Master </a> is <br></span></li><li>
      <span style="line-height:1.6;">The <a href=/have-a-definition-of-ready>Definition of Ready </a> and the <a href=/done-do-you-go-beyond-done-and-follow-a-definition-of-done>Definition of Done </a></span></li><li>
      <span style="line-height:1.6;"><span style="line-height:20.8px;">W</span><span style="line-height:20.8px;">hen the <a href=/methodology-do-you-do-daily-scrums-aka-stand-up-meetings>daily standups​</a> occur and when the next sprint review is scheduled</span><span style="line-height:20.8px;"> </span></span></li><li>
      <span style="line-height:1.6;">The current sprint backlog</span></li><li>
      <span style="line-height:1.6;">Show the current build status</span></li><li>
      <span style="line-height:1.6;">Show links to:</span></li><ul><li><span style="line-height:1.6;">Staging environment</span></li><li>
         <span style="line-height:1.6;">Production environment</span></li><li>
         <span style="line-height:1.6;">Any other external service used by the project e.g. Octopus Deploy, Application Insights, RayGun, Elmah, Slack</span></li></ul></ol><p>Your solution should also contain the standard <a href="/Pages/DoYouMakeInstructions.aspx">_Instructions.docx</a> to your solution file for additional details on getting the project up and running in Visual Studio.</p><p>For particularly large and complex projects you can use an induction tool like <a href="https://www.sugarlearning.com/">SugarLearning</a> to create a course for getting up to speed with the project.</p><dl class="image"><dt><img src="2016-06-06_7-18-43.png" alt="2016-06-06_7-18-43.png" style="width:800px;" /><br></dt></dl><br>
